## 5. Internal Linking Rules

For every new blog post published:
1. Link to the relevant **pillar page** from the blog post body (1-2 contextual links)
2. Link to **2-3 relevant safari packages** from the blog post body
3. Link to **1-2 relevant destination pages** from the blog post body
4. Add the blog post to the **sidebar "Recent Posts"** widget (automatic via controller)
5. Ensure the blog post's **tags and category** match safari/destination keywords for the automatic related content system

The internal linking controller in [`BlogController.php`](app/Http/Controllers/BlogController.php:42) already handles automatic matching of blog tags/category/title to safaris and destinations — this will work automatically once posts have relevant tags.

## 6. Schema & Meta Tag Template for Blog Posts

When creating each blog post in the admin panel, use these defaults:

```php
// SEO Title format:
"{Primary Keyword} | Golden Memories Safaris Blog"

// SEO Description format:
"{Compelling summary with primary + secondary keyword, under 155 chars}"

// Meta Keywords:
"{primary keyword}, {secondary keyword 1}, {secondary keyword 2}, Tanzania safari, Golden Memories Safaris"

// Category:
// Use one of: "Safari Guide", "Serengeti", "Kilimanjaro", "Zanzibar", "Travel Tips", "Wildlife"
```

The Article schema in [`blog-detail.blade.php`](resources/views/blog-detail.blade.php:27) will automatically use the post title, description, author, and dates.
