# SEO Audit Report: Golden Memories Safaris (gmsafaris.co.tz)

**Audit Date:** 2026-05-05  
**Auditor:** SEO Lead  
**Site URL:** https://www.gmsafaris.co.tz  
**Tech Stack:** Laravel (PHP) + Blade Templates (server-side rendered)  
**Target Markets:** UK, US, Germany, France, Netherlands, Italy, Spain, Canada, Australia, Japan, China, Singapore, UAE  

---

## Executive Summary

Golden Memories Safaris has a **solid foundation** for SEO success: server-side rendered Blade templates (excellent for crawlability), a clean URL structure via Laravel routes, JSON-LD structured data on most pages, and a well-organized layout with yield sections for meta tags. However, there are **critical issues** preventing the site from being indexed and ranked effectively: no XML sitemap, no robots.txt, broken contact forms, missing canonical/OG tags on key pages, and no hreflang implementation despite targeting 13+ countries.

**Estimated traffic impact if all Critical issues are fixed:** 3-5x increase in indexed pages within 4-6 weeks, enabling the site to compete for the ~60% informational search intent queries that currently go unserved.

## 1. TECHNICAL SEO

### 1.1 XML Sitemap — ❌ MISSING
**Severity:** 🔴 Critical  
**Location:** No `sitemap.xml` found in [`public/`](gmsafaris-laravel/public/)

**Issue:** There is no XML sitemap. Google cannot efficiently discover all pages, especially the dynamically generated safari, destination, and blog detail pages.

**Impact:** Without a sitemap, Googlebot must rely entirely on internal links to discover pages. Deep or less-linked pages may never get indexed. This is the #1 reason the site has only ~10 monthly organic visits.

**Fix:** Generate a dynamic sitemap covering:
- Static pages: `/`, `/about`, `/contact`, `/gallery`, `/booking`, `/testimonial`, `/service`, `/besttimetovisit`, `/visa`, `/kilimanjaroroutes`, `/tailoredsafaris`, `/luxurysafari`, `/mountaintrekking`, `/groupsafaris`, `/budgetsafari`, `/zanzibarbeachholiday`, `/localcustoms`, `/healthandsafety`, `/insuarance`, `/privacypolicy`, `/termsandconditions`
- Dynamic safaris: `/safari/{slug}` (from DB)
- Dynamic destinations: `/destination/{slug}` (from DB)
- Dynamic blog posts: `/blog/{slug}` (from DB)
- Blog listing: `/blog`
- Safaris listing: `/safaris`
- Destinations listing: `/destinations`

**Implementation:** Create a dedicated [`SitemapController`](gmsafaris-laravel/app/Http/Controllers/) or use a package like `spatie/laravel-sitemap`. Register route `/sitemap.xml` that returns XML with proper `<lastmod>`, `<changefreq>`, `<priority>`.

### 1.2 Robots.txt — ❌ MISSING
**Severity:** 🔴 Critical  
**Location:** No `robots.txt` found in [`public/`](gmsafaris-laravel/public/)

**Issue:** No robots.txt exists. While this doesn't block crawling, it means:
1. No sitemap reference for Googlebot
2. No crawl budget directives (e.g., disallow admin paths)
3. Google may crawl admin/login pages unnecessarily

**Fix:** Create [`public/robots.txt`](gmsafaris-laravel/public/robots.txt):
```
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /login
Disallow: /register
Disallow: /password-reset

Sitemap: https://www.gmsafaris.co.tz/sitemap.xml
```

### 1.3 Hreflang Tags — ❌ MISSING
**Severity:** 🔴 Critical  
**Location:** [`layouts/app.blade.php`](gmsafaris-laravel/resources/views/layouts/app.blade.php:2)

**Issue:** The site targets 13+ countries/markets (UK, US, Germany, France, Netherlands, Italy, Spain, Canada, Australia, Japan, China, Singapore, UAE) and has a GTranslate widget offering EN, IT, PL, SV, FR. However, there are **zero hreflang tags** in the `<head>`. Google cannot determine which language/region version to serve.

**Impact:** Users in Germany (a key market per `.seo-context.md`) searching in German will not see this site because there's no German language version or hreflang signal. The GTranslate widget is client-side JavaScript — Googlebot does not execute it, so it sees only the English version.

**Fix:** Add hreflang tags to [`layouts/app.blade.php`](gmsafaris-laravel/resources/views/layouts/app.blade.php) in the `<head>`:
```html
<link rel="alternate" hreflang="en" href="https://www.gmsafaris.co.tz{{ request()->getPathInfo() }}" />
<link rel="alternate" hreflang="en-US" href="https://www.gmsafaris.co.tz{{ request()->getPathInfo() }}" />
<link rel="alternate" hreflang="en-GB" href="https://www.gmsafaris.co.tz{{ request()->getPathInfo() }}" />
<link rel="alternate" hreflang="x-default" href="https://www.gmsafaris.co.tz{{ request()->getPathInfo() }}" />
```
**Note:** True hreflang requires separate URLs per language. If using GTranslate's SEO mode (subdirectories like `/it/`, `/fr/`), enable it. Otherwise, consider creating dedicated language subdirectories or subdomains for key markets (German, French, Italian).

3. Add CSRF token `@csrf` to all forms
4. Wire up [`InquiryController`](gmsafaris-laravel/app/Http/Controllers/InquiryController.php) (already exists) to handle the logic
